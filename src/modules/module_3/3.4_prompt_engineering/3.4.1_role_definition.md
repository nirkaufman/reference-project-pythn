## The Prompt Engineer (The Domain Expert)
### Owns the AI's behavior, logic, and output format.
1. Works in the UI: Uses tools like the LangSmith playground—no code required.
2. Crafts Instructions: Designs system and user prompts to guide the AI's tone and decision-making.
3. Enforces Contracts: Ensures the prompt strictly outputs the format the application needs (e.g., clean JSON).
4. Iterates & Tests: Evaluates edge cases against test datasets to fix hallucinations.
5. Versions & Deploys: Commits "golden" prompts to a centralized hub for the developers to use.

## The Developer (The Application Architect)
### Owns the system integration, applying Traditional Engineering principles.
1. Works in the Codebase: Uses SDKs (like LangChain) in Python or TypeScript.
2. Pulls Prompts via API: Fetches the latest version-controlled prompt by its name/ID, ignoring the natural language text inside it.
3. Injects Context: Writes the logic to grab live system data (database queries, webhooks) and maps it to the prompt's variables.
4. Manages Execution: Handles the actual API calls to the LLM (managing latency, rate limits, and retries).
5. Parses & Routes: Takes the structured output (JSON) provided by the LLM and integrates it into the core application logic.

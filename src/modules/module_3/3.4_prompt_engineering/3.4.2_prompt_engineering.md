Phase 1: The Prompt Engineer (LangSmith UI)
In the LangSmith UI, the prompt engineer will craft the instructions to ensure the AI acts as a strict data parser.

### Create the Prompt
Name the prompt something like sales-lead-extractor.
__System Message__  

```text
You are an expert data extraction assistant. Your task is to analyze post-call summaries generated by our AI voice agents and extract structured information for the CRM.
Extract the following data points and output them strictly as a JSON object:
• "customer_name": The full name of the potential customer.
• "company": The customer's company name.
• "meeting_date": The agreed-upon date and time for the meeting.
• "address": The customer's physical address or office location.
• "pain_points": A brief summary of their current challenges or needs.
• "budget": The mentioned budget, if any.
• "action_items": Anything the sales rep needs to prepare or bring to the meeting.
If a specific piece of information is missing from the summary, output null for that field.
Output ONLY the JSON object. Do not include markdown formatting like ```json.
User Message
Please extract the data from the following call summary:
{call_summary}
```

### The Developer (SDK/Code)
Now, the developer pulls this structured prompt. Because the prompt engineer ensured the output is strictly JSON, the developer can immediately parse it into a standard language dictionary or object, demonstrating robust, predictable application behavior.

```python
# Example:
import json
from langsmith import Client
from openai import OpenAI
from langchain_core.messages import convert_to_openai_messages

client = Client()
oai_client = OpenAI()

# 1. Developer pulls the latest prompt version
pulled_prompt = client.pull_prompt("sales-lead-extractor")

# 2. In a real app, this summary string comes directly from the 11labs webhook/API response
mock_11labs_summary = """
Outbound call connected with Elena Rodriguez, VP of Engineering at CloudScale Inc. Elena mentioned they are actively looking for a new cloud infrastructure provider because their current solution is causing severe latency issues during peak hours, and their budget for Q3 is around $15k/month. I proposed a discovery meeting with our senior sales rep, David. Elena agreed to an in-person meeting next Tuesday, May 5th at 2:00 PM. She asked if David could bring a technical whitepaper on our failover protocols. The meeting will be at her office located at 890 Tech Boulevard, Building B, Seattle, WA. Call ended positively.
"""

# 3. Format the prompt with the variable
formatted_prompt = pulled_prompt.invoke({
    "call_summary": mock_11labs_summary
})

# 4. Execute the call
response = oai_client.chat.completions.create(
    model="gpt-4o", 
    messages=convert_to_openai_messages(formatted_prompt.messages),
    temperature=0 # Best practice: set temperature to 0 for data extraction tasks
)

# 5. Parse the output directly into a Python dictionary
raw_output = response.choices[0].message.content
extracted_data = json.loads(raw_output)

# Now the developer can seamlessly insert this into a CRM database
print(f"Booking meeting for {extracted_data['customer_name']} at {extracted_data['address']}")
print(f"Don't forget to prepare: {extracted_data['action_items']}")
```

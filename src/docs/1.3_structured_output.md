### What is Structured Output?
Structured output is the practice of forcing an LLM to respond in a predictable, machine-readable format—most commonly
JSON, but also YAML or specific Markdown schemas. Instead of a "chatty" paragraph, you get a data object.

### Key Benefits

#### Reliable Programmatic Integration
If your Python code needs to extract a "Recipe Name" to save it to a database, you cannot rely on regex to find it 
in a wall of text. Structured JSON allows you to use response['dish_name'] with 100% certainty.

#### Tool Use & Function Calling
For agents to actually do things (like searching the web or ordering groceries), they must output data that
matches a specific function signature. Structured output is the bridge between the LLM's 
"thinking" and the code's "execution."

#### UI Consistency
As you discuss in your "Death to the Loading Spinner" philosophy, structured output allows the 
frontend to know exactly which components to render (e.g., an IngredientList component vs. a StepByStep component) 
as soon as the first few tokens of the JSON object stream in.

#### Remember
we don't want a "creative" response.
we want a response that adheres to the 
contract we've defined in our code.
